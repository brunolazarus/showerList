# Devlog — 2026-03-22

## Overview

Completed phases 8.1, 9.1, and 10.1 — the simplified alternative track that
replaces voice-activation-window-based control with direct clap-to-command
mapping, wired end-to-end into the Electron tray app and Spotify.

---

## Phase 8.1 — Direct Clap Command Mapping

Implemented `clapDetector.ts` with:
- **Single clap** → pause/play toggle (state tracked in voice process)
- **Double clap** (150–600ms gap) → skip
- RMS spike detection (8× EMA baseline) + spectral centroid check (>2kHz) to
  filter water/low-frequency noise
- 2-second cooldown after any fired event

Wired into `index.ts` with explicit pause/play tracking.

**M3 Mac issue:** CoreAudio only exposes a mono mic input. Added `remix 1 1`
to the `rec` command so sox upmixes to stereo before streaming to the pipeline.
Solved via a macOS restart (permissions were stuck) + the remix fix.

**Debug added:** periodic RMS/baseline/threshold logging to stderr so we can
tune SPIKE_FACTOR if the shower environment proves too noisy.

---

## Phase 9.1 — ASR Scaffold + Ready Handshake

Created `apps/voice/src/recognition/`:
- `aliases.ts` — single source of truth for command vocabulary
- `commandMatcher.ts` — normalize → exact word → substring → Levenshtein ≤1
- `asr.ts` — Whisper tiny.en via `@xenova/transformers` (local, no API calls,
  ~80MB, caches in `VOICE_MODEL_CACHE`)

ASR is scaffolded but not yet the primary command path — claps cover the main
use case (pause/play/skip). Voice recognition remains available for future
extension (e.g. `previous`, which claps can't cover with the current pattern).

Added **`"ready"` IPC state** to `ipc.ts` — emitted on the first captured
audio frame. This gives `voiceManager` a reliable signal that the pipeline is
live before the desktop considers the process ready.

---

## Phase 10.1 — Desktop Integration

Connected the voice process to the Electron tray app:

**`settingsStore.ts`** — persists voice enabled/disabled to `userData/settings.json`.

**`voiceManager.ts`** — spawns `apps/voice/dist/index.js`, reads stdout
line-by-line as JSON, resolves `start()` on the `"ready"` handshake, fires
typed callbacks for commands and status changes. `stop()` sends SIGTERM.

**`index.ts`** (desktop):
- Voice starts automatically on launch if enabled
- Tray tooltip reflects voice state (Listening…, Processing…)
- New tray menu item "🎙 Voice: On/Off" to toggle at runtime

**`spotifyService.ts`** — added `cmdPause()` and `cmdPlay()` for discrete
command dispatch (separate from the existing `cmdToggle()`).

**Tests:** 11 new tests for `voiceManager` and `settingsStore`.
Full suite: **85 tests, all passing**.

---

## Notes & Known Risks

- **Shower noise:** SPIKE_FACTOR of 8× and spectral centroid >2kHz should
  filter water noise, but real-world testing in a shower is still needed.
  Main tuning knobs: `SPIKE_FACTOR` and `CENTROID_MIN_HZ` in `clapDetector.ts`.
- **Voice path in production:** `voiceManager` resolves the voice app path
  relative to `__dirname`. Packaging with `electron-builder` will require
  the voice app to be bundled as an extra resource.
- **`previous` command:** not reachable via claps yet. Triple-clap or voice
  is the natural path — deferred to a future phase.
