## Decisions & Rationale

| Decision                                         | Rationale                                                                       |
| ------------------------------------------------ | ------------------------------------------------------------------------------- |
| `showerlist://` URL scheme over `localhost` HTTP | No port conflicts, no ephemeral server, RFC 8252 recommended for native apps    |
| No client secret                                 | PKCE is sufficient for public clients; eliminates secret management entirely    |
| `safeStorage` for tokens                         | OS keychain integration, survives reinstall, no plaintext on disk               |
| Monorepo (pnpm workspaces)                       | Keeps desktop and voice pipeline cleanly separated; shared types in `packages/` |
| Strict TypeScript + `Result<T,E>`                | Matches CLAUDE.md conventions; forces explicit error handling                   |

## Phase 2 — PKCE OAuth Flow

**Session date:** 2026-03-17 (continued)
**Branch:** `feature/phase-2-oauth`
**Role:** Implementer

### Work Completed

#### `apps/desktop/src/main/oauthManager.ts` (new file)

- `generateCodeVerifier()` — 32 random bytes → base64url, no padding, using Node `crypto`
- `generateCodeChallenge(verifier)` — SHA-256 → base64url via `crypto.createHash`
- `buildAuthUrl(params)` — constructs full `https://accounts.spotify.com/authorize` URL with
  the three approved scopes (`user-modify-playback-state`, `user-read-playback-state`,
  `user-read-currently-playing`)
- `startAuth(clientId)` — generates PKCE pair + random hex `state`, stores in module-level
  `pendingAuth`, validates URL prefix before `shell.openExternal`, sets 5-min timeout
- `handleCallback(url)` — validates `showerlist:` scheme, extracts `code`+`state`,
  CSRF-guards state match, POSTs token exchange to `accounts.spotify.com/api/token`,
  calls `tokenStore.saveTokens`, returns `Result<void>`
- `refreshAccessToken()` — stub, throws `"Not implemented — Phase 4"`
- `setAuthTimeoutHandler()` — lets `index.ts` register a tray tooltip callback on timeout

#### `apps/desktop/src/main/index.ts` (modified)

- `app.setAsDefaultProtocolClient('showerlist')` called once in `whenReady`
- `app.on('open-url')` — macOS deep-link handler → `handleCallback` → `refreshMenu`
- `app.on('second-instance')` — Windows/Linux argv fallback → same path
- "Connect to Spotify…" click wired to `startAuth(clientId)` (reads `SPOTIFY_CLIENT_ID` from env)
- Menu item disabled when already connected
- Auth timeout surfaces via `tray.setToolTip`

#### `utils/` (new folder)

- `trayIcon.svg` — shower head + water drops + eighth note SVG source (18×18 viewBox)
- `gen-icon.js` — Node script that generated the initial placeholder PNG

#### `apps/desktop/assets/`

- `trayIconTemplate.png` — 26×26 rendered from SVG (1x)
- `trayIconTemplate@2x.png` — 52×52 rendered from SVG (2x Retina)

### Security Properties Verified

| Invariant | How enforced |
| --------- | ------------ |
| `shell.openExternal` URL validation | Guard throws if URL doesn't start with `https://accounts.spotify.com/authorize` |
| CSRF protection | `state` param matched against `pendingAuth.state` before token exchange |
| Tokens never logged | Flow only through `tokenStore.saveTokens` → `safeStorage` |
| No new npm packages | Only Node builtins (`crypto`, `fetch`) and existing Electron APIs |
