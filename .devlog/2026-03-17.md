## Decisions & Rationale

| Decision                                         | Rationale                                                                       |
| ------------------------------------------------ | ------------------------------------------------------------------------------- |
| `showerlist://` URL scheme over `localhost` HTTP | No port conflicts, no ephemeral server, RFC 8252 recommended for native apps    |
| No client secret                                 | PKCE is sufficient for public clients; eliminates secret management entirely    |
| `safeStorage` for tokens                         | OS keychain integration, survives reinstall, no plaintext on disk               |
| Monorepo (pnpm workspaces)                       | Keeps desktop and voice pipeline cleanly separated; shared types in `packages/` |
| Strict TypeScript + `Result<T,E>`                | Matches CLAUDE.md conventions; forces explicit error handling                   |
