#### Hardening: stream error handling (`capture.ts`)

`startCapture` previously registered no `'error'` listener on the audio stream.
A mic unavailability or mid-session drop would propagate as an uncaught Node.js error
and crash the process with no structured IPC output.

Fix: added `onError?: (err: Error) => void` parameter; `s.on('error', …)` routes failures
into the existing `VoiceStatusMessage { type: 'status', state: 'error' }` IPC path.

---

#### Hardening: shutdown race condition (`capture.ts`, `index.ts`)

`stopCapture()` previously called `stream.quit()` (async portaudio teardown) and then
`process.exit(0)` synchronously. The audio device could be left locked on some platforms
because teardown was not awaited.

Fix: `stopCapture(onDone?: () => void)` passes `onDone` as the quit callback, so
`process.exit(0)` is deferred until portaudio completes.

---

#### Hardening: RingBuffer frame-size guard (`ringBuffer.ts`)

`push(frame)` previously called `buf.set(frame, offset)` without validating `frame.length`.
An oversized frame would silently overwrite the next slot — latent corruption with no
signal to the caller.

Fix: added a `RangeError` guard before the write. Any frame with the wrong element count
throws immediately rather than corrupting the buffer.

---

#### Dependency swap: naudiodon → sox (`capture.ts`, `voice/package.json`, `desktop/package.json`)

**Root cause:** `naudiodon` bundles `libportaudio.dylib` with only `x86_64`/`i386` slices.
The `naudiodon.node` binary is built for `arm64` (Apple Silicon). When the dylib is
`dlopen`-ed at runtime, `Pa_Initialize()` inside `PaContext::PaContext` dereferences a
null pointer and raises `SIGSEGV`. This is not catchable in JavaScript regardless of
try/catch or lazy-load strategy.

**Fix:** replaced with `child_process.spawn('rec', […])` (sox's record command):
- sox is shipped as an arm64-native Homebrew bottle — no recompilation needed.
- Streams raw 16-bit signed PCM at 48 kHz stereo to stdout; frame accumulation and
  `Int16Array` parsing logic is unchanged.
- Zero npm native-addon dependencies.
- A failing `rec` spawn (ENOENT, non-zero exit) surfaces through the `onError` callback
  as a structured IPC `{ type: 'status', state: 'error', detail: … }` message.
- Removed `electron-rebuild` scripts and devDependency from `desktop/package.json`
  (they existed solely to recompile naudiodon for Electron's Node ABI).
- Added `brew install sox` prerequisite to `RUNBOOK.md`.

---

#### Hardening: deterministic error exit (`index.ts`)

Consolidated two error paths (synchronous throw from `startCapture` + async `onError`
callback) into one `onCaptureError` helper that:
1. Emits `{ type: 'status', state: 'error', detail }` to stdout.
2. Sets `process.exitCode = 1`.
3. Calls `process.exit(1)` via `setImmediate` — gives stdout one event-loop tick to
   flush the IPC message before the process terminates, so the parent reliably receives
   the structured error before the non-zero exit code.

## Technical Debt

### Cross-platform audio capture

sox (`rec`) works on macOS with `brew install sox`. It is not a portable solution:

| Platform | Status |
| -------- | ------ |
| macOS (Apple Silicon) | ✅ Works via `brew install sox` |
| macOS (Intel) | ✅ Works via `brew install sox` |
| Linux | ❌ sox/rec available but not validated; requires `apt install sox` and ALSA/PulseAudio config |
| Windows | ❌ sox binaries available but paths, device names, and default input flags differ |

Before any Linux or Windows target is needed, `capture.ts` must be revisited.
Options at that point:
- Conditional spawn command (platform sniff + appropriate args for each OS)
- Switch to a proper cross-platform native binding (e.g. `node-audiorecorder`, a
  rebuilt `naudiodon` with platform-specific portaudio, or `soundio` bindings)

**Tracked as:** tech debt — Phase 6 is macOS-only.
