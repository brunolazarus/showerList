### Phase 3 — `packages/spotify-client` REST Wrapper

**New package — zero Electron dependency, pure Node/fetch.**

#### `packages/spotify-client/package.json`

- Workspace package named `@showerlist/spotify-client`
- `build` script (`tsc`) produces `dist/` consumed by desktop app
- `typecheck` script for `pnpm -r typecheck` integration

#### `packages/spotify-client/tsconfig.json`

- Extends `../../tsconfig.base.json`; `rootDir=src`, `outDir=dist`

#### `packages/spotify-client/src/types.ts`

- `SpotifyErrorCode` — discriminated union: `unauthorized | premium_required | no_active_device | network_error | unknown`
- `SpotifyError` — `{ code, message }`
- `Result<T, E>` — `{ ok: true; value: T } | { ok: false; error: E }`
- `TrackInfo` — `{ name, artists: string[], isPlaying }`
- `PlayerState` — `{ isPlaying, track: TrackInfo | null }`

#### `packages/spotify-client/src/index.ts`

Shared internal helpers:
- `mapStatus(status)` — maps 401/403/404/other to typed `SpotifyError`
- `apiRequest(token, method, endpoint)` — single `fetch` call with Bearer auth, targets `https://api.spotify.com` only
- `controlCommand(token, method, endpoint)` — wraps `apiRequest`, catches network errors, returns `Result<void, SpotifyError>`
- `parseTrackInfo(body)` — maps Spotify JSON to `TrackInfo | null`

Exported functions:
- `next(token)` — `POST /v1/me/player/next`
- `previous(token)` — `POST /v1/me/player/previous`
- `pause(token)` — `PUT /v1/me/player/pause`
- `play(token)` — `PUT /v1/me/player/play`
- `getPlayer(token)` — `GET /v1/me/player`; 204 → `{ isPlaying: false, track: null }`
- `getCurrentTrack(token)` — `GET /v1/me/player/currently-playing`; 204 → `null` (nothing playing)

Type re-exports: `Result`, `SpotifyError`, `SpotifyErrorCode`, `TrackInfo`, `PlayerState`

#### `packages/spotify-client/src/index.test.ts`

- 40 tests covering all 6 functions

- `getCurrentTrack`: 204 → null, multi-artist parsing, 401 error, network_error

### Phase 4 — Desktop Integration and Token Refresh

#### `apps/desktop/src/main/oauthManager.ts` (modified)

- `refreshAccessToken(clientId, refreshToken)` — fully implemented (was a stub):
  - POST `grant_type=refresh_token` to `accounts.spotify.com/api/token`
  - Handles optional token rotation (Spotify may or may not return a new `refresh_token`)
  - Validates response shape; saves via `tokenStore.saveTokens`
  - Returns `Result<TokenData>`

#### `apps/desktop/src/main/spotifyService.ts` (new)

Stateful wrapper around `spotify-client` for the main process:

- `init(tokens, clientId)` — loads token state into module; called at startup
- `isConnected()` / `disconnect()` — connection state + keychain clear
- `ensureFreshToken()` — refreshes proactively when `expiresAt - now ≤ 2 min`
- `doRefresh()` — single-flight guard via `refreshPromise`; on failure clears tokens
- `run(fn)` — ensures fresh token, calls `fn(accessToken)`, on 401 does one refresh + one retry
- `cmdNext()`, `cmdPrevious()`, `cmdToggle()`, `fetchCurrentTrack()` — public command surface
  - `cmdToggle` reads `getPlayer` first to determine pause vs play

#### `apps/desktop/src/main/index.ts` (modified)

- **Startup**: loads stored tokens → `spotifyService.init()` → `startTitlePolling()` → `updateTrayTitle()`; skips if no `SPOTIFY_CLIENT_ID` or no stored tokens
- **Tray actions**: Previous / Pause-Play / Skip wired to `spotifyService` commands; errors routed to `handleCommandError`
- **`handleCommandError(error)`**: surfaces `no_active_device`, `premium_required`, and `unauthorized` (→ disconnect + require reconnect) as tray tooltip messages
- **`updateTrayTitle()`**: calls `fetchCurrentTrack`; sets `tray.setTitle("Artist — Track")` or clears it
- **`startTitlePolling()` / `stopTitlePolling()`**: 30-second `setInterval`; guarded against double-start
- **`scheduleTrackRefresh()`**: 500ms debounce fired after every command to update track state
- **`onAuthSuccess()`**: replaces inline `refreshMenu` in both deep-link handlers; inits service + starts polling after new auth

#### Package wiring

- `apps/desktop/package.json`: added `@showerlist/spotify-client: workspace:*` dependency; `dev` now builds spotify-client before `tsc`
- `package.json` (root): `build-spotify` script; `typecheck` builds `dist/` before running `tsc --noEmit`
